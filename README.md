# Nestograms 🧩
> A highly polished, minimalist, offline-first Nonogram Chrome Extension (Manifest V3) built with Vanilla TypeScript, Web Components, Vite, CRXJS, and Firebase Firestore.

---

## 🎨 Aesthetic & Design Philosophy
Nestograms adheres strictly to a sleek, modern **"Off-Whites and Off-Blacks"** aesthetic:
- **Warm Off-White Backgrounds** (`#faf9f6`, `#f0efe9`) paired with **Soft Off-Black Text & Borders** (`#1a1a18`).
- **Zero Generic Colors**: Subtle warm accents, HSL-tailored indicators, and clean line-art SVG iconography.
- **Responsive Layout**: Scales dynamically up to 20x20 grids without text clipping or UI distortion.
- **Thick Grid Dividers**: Every 5th grid line is rendered with darker, thicker borders to aid visual tracking.

---

## 🏗️ Architectural Overview

### 1. Pure Engine Logic (`src/engine/`)
- Completely decoupled from DOM and browser side effects.
- **Scoring (`scoring.ts`)**: Base 1000 pts with time decay, 15% heart penalty per error, and strict tier multipliers ($1\times$, $1.5\times$, $2\times$, $3.5\times$).
- **Grid Mathematics (`grid.ts`)**: Algorithmic clue generation, line satisfaction verification, and completion checking.
- **Animation Engine (`animation.ts`)**: Diagonal Manhattan-distance wave calculation and custom vanilla 2D physics confetti particle cannons.
- **Anti-Cheat (`anticheat.ts`)**: Cryptographic SHA-256 state hashing combining grid state, score, elapsed time, and puzzle ID.

### 2. Offline-First Sync & Storage (`src/services/`)
- **Storage Service (`storage.ts`)**: Dual-mode data persistence using Chrome `chrome.storage.local` with fallback to `localStorage` when testing outside extension context.
- **Firebase Sync (`firebase.ts`, `sync.ts`)**:
  - Offline-first queuing: Scores recorded while offline are queued locally and automatically retried upon re-establishing network connectivity.
  - Plausibility & Rate Limiting: Minimum completion time threshold check ($0.2\text{s/cell}$) and 30-second submission throttling per UID.
  - Anonymous Authentication: Auto-initializes Firebase Anonymous Auth for seamless leaderboard identity.

### 3. Native Web Components (`src/components/`)
- **`<nesto-app>`**: Root container featuring hardware-accelerated CSS view transitions and a modal intercept (`<nesto-modal>`) to prevent accidental data loss mid-game.
- **`<nesto-player>`**: Exclusive keyboard game controls (auto-repeat timers, simultaneous $Z$ and $X$ priority resolution), focus trapping, and window blur protection that hides the grid behind a pause overlay during tab switching.
- **`<nesto-tutorial>`**: Interactive 4x4 sandbox tutorial launched automatically on first install with blinking guidance tips and fake score calculations.
- **`<nesto-leaderboard>`**: Infinite scroll global leaderboard with CSS sticky rank highlighting for the active player's row.
- **`<nesto-settings>`**: Profile customization, 1-to-1 keybind remapping with modifier support, and custom popup vs. windowed display mode switching.

## 🚀 Getting Started

### Prerequisites
- [Node.js](https://nodejs.org/) (v18 or v20+ recommended)
- [npm](https://www.npmjs.com/)

### 1. Installation & Build
```bash
# Clone repository and navigate to root
cd nestograms

# Install dependencies
npm install

# Run Vitest unit tests (engine math, scoring, grid validation)
npm test

# Run ESLint formatting check
npm run lint

# Build extension bundle for production (outputs to /dist)
npm run build
```

### 2. Loading into Google Chrome
1. Open Google Chrome and navigate to `chrome://extensions/`.
2. Enable the **Developer mode** toggle in the top-right corner.
3. Click **Load unpacked** in the top-left corner.
4. Select the `dist/` directory generated by `npm run build`.
5. Pin **Nestograms** to your Chrome toolbar!

## ⚙️ Firebase Setup & Database Seeding
To connect Nestograms to a live Firebase backend for global score synchronization and cloud puzzles:

1. Create a project in the [Firebase Console](https://console.firebase.google.com/).
2. Enable **Firestore Database** (in native mode) and **Authentication** (enable **Anonymous Auth** provider).
3. Copy `.env.example` to `.env` in the root directory and populate your credentials:
   ```env
   VITE_FIREBASE_API_KEY="your-api-key"
   VITE_FIREBASE_AUTH_DOMAIN="your-project.firebaseapp.com"
   VITE_FIREBASE_PROJECT_ID="your-project"
   VITE_FIREBASE_STORAGE_BUCKET="your-project.appspot.com"
   VITE_FIREBASE_MESSAGING_SENDER_ID="123456789"
   VITE_FIREBASE_APP_ID="1:123456789:web:abcdef"
   ```
4. Deploy Security Rules by pasting the contents of `firestore.rules` into your Firebase Console under **Firestore Database -> Rules**.
5. **Seed Initial Puzzles**: Run the automated seeding script to populate Firestore with sample puzzles:
   ```bash
   npx tsx scripts/seed-puzzles.ts
   ```

## 🕹️ Game Controls
| Action | Keybind (Default) | Notes |
| :--- | :--- | :--- |
| **Move Selector** | `Arrow Keys` | Continuous auto-repeat on hold; wraps around grid boundaries. |
| **Fill Cell** | `Z` | Holding allowed; overwrites marked `X` cells. |
| **Mark Empty** | `X` | Takes priority if pressed simultaneously with `Z`. |
| **Clear Cell** | `Backspace` / `Delete` | Resets cell to empty state. |
| **Resume Game** | `Click` / `Any Key` | Dismisses the blur protection overlay. |

## 📦 CI/CD & Automated Release Pipeline
The `.github/workflows/release.yml` workflow automatically executes whenever a version tag (`v*`) is pushed:
1. Runs full linter and Vitest test suites.
2. Compiles and bundles production code via Vite + CRXJS.
3. Generates a Web Store ready `.zip` payload (`nestograms-vX.Y.Z-webstore.zip`).
4. Generates a compiled `.crx` file.
5. Publishes a GitHub Release complete with source archives and automated changelogs.
